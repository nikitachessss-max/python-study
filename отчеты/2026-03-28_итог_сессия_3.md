## Что реально усвоено (могу написать сам или объяснить словами)

- **Создание списка**, обращение по индексу `[0]`, `[1]`, замена элемента.
- **`len`** — сколько элементов; последний индекс = `len - 1`.
- **Срез** `[a:b]` — правая граница не входит.
- **`for` по списку** — в переменную попадает элемент, не индекс.
- **`append`** — дописать одно значение в конец списка.
- **`del`** — убрать по индексу; **`pop()`** — убрать последний и можно сохранить.
- **`while True` + `break`** — бесконечный цикл с выходом изнутри; разница с `while word != ...` понятна (один `input` вместо двух).
- **`range(1, n + 1)`** — нумерация с единицы; правая граница не входит.

## Что понято на уровне «читаю и понимаю, но сам с нуля не напишу»

- **f-строка внутри цикла для сборки строки чека** (`line = f"{i}. {product} — {cost} руб."`): смысл понятен при разборе, но собрать самостоятельно без опоры на пример — пока тяжело.
- **`lines.append(line)`** после сборки строки: понимание «зачем» есть, но при написании с нуля забывается или путается с `print`.
- **Связка `n → range(1, n + 1) → тело цикла → итоги после цикла`**: логика разобрана, но при самостоятельном написании челленджа потребовалось построчное ведение или подглядывание.
- **`if len(lines) >= 3:` + срез `[:3]`**: понятно при чтении, не очевидно при написании.

## Что пока НЕ усвоено / слабые места

- **Самостоятельное написание программы «с нуля» на 15+ строк без опоры на эталон** — пока не получается. Челлендж 1–3 без подсказок не был написан; кафе и курьер — с подглядыванием.
- **Путаница `if` / `else` / `elif`**: в кафе написал `else sale == "нет":` вместо `else:` — синтаксис условий при самостоятельном письме ещё плавает.
- **Кавычки внутри строки**: дважды ломал синтаксис, вставляя `"` внутрь `"..."`.
- **Переменная цикла `for`**: понимание есть, но при первом написании `append` забыл добавить `.append()` после `input` — положил ввод в переменную, но не в список.
- **Мотивация и усталость**: к концу сессии явно устал; фразы «всё забыл», «не в силах» — реальное состояние, не лень.

## Честная общая оценка

**Прогресс есть, и он настоящий.** За 10 дней (три учебных захода) ученик прошёл от «индекс с нуля» до рабочего чека с циклом, списком, срезом и `while`. Это не мало.

**Но.** Большая часть сложного кода (челлендж 1–3, чек без `n`) была **написана или дописана агентом**, а не учеником. Ученик **читает** и **понимает** при разборе, но **воспроизвести** с нуля без опоры на готовый пример пока **не может**.

**Это нормально для текущего уровня.** Навык «пишу сам» формируется через повторение одинаковых каркасов, а не через «один раз увидел — запомнил». Кафе и курьер написаны с подглядыванием — и это уже шаг вперёд по сравнению с «напиши за меня».

**Слабые точки для контроля перед сессией 4:**
1. Синтаксис `if` / `elif` / `else` — путается при самостоятельном письме.
2. Кавычки в строках — дважды `SyntaxError`.
3. Связка «пустой список → цикл → append → использовать список после цикла» — понимание есть, автоматизма нет.
4. Длинные программы (>10 строк кода) — теряется без пошаговой опоры.

**Рекомендация:** в начале сессии 4 дать одну разминку на знакомый каркас (пустой список + цикл + append + for по списку) с **новым сюжетом**, **без подглядывания**, чтобы проверить, сел ли шаблон. Если нет — ещё одна итерация, потом словари.
